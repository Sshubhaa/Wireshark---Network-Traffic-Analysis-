# Wireshark---Network-Traffic-Analysis

## About This Project
This project is part of my hands-on cybersecurity learning journey as I transition into the field.

It is based on a TryHackMe Wireshark lab, focused on capturing, filtering, and analyzing network traffic to understand how data flows across networks.

## Objectives
- Understand how network packets are structured  
- Capture and analyze live traffic using Wireshark  
- Learn how to filter traffic for meaningful analysis  
- Identify key protocols and potential security insights 

## Tools & Technologies
- Wireshark
- Kali Linux
- Networking basics

## Key Concepts Covered
- Packet capture and inspection
- Protocol analysis (TCP, HTTP, DNS)
- Filtering and searching traffic

  ## Practical Tasks Performed
- Captured live network traffic
- Applied filters to isolate specific protocols
- Inspected packet details such as source, destination, ports
- Identified unencrypted data in HTTP traffic

## Key Learnings
- Network traffic can expose sensitive information if not encrypted
- Filtering is essential when analyzing large packet captures
- Understanding protocols helps identify abnormal or suspicious behavior
- Wireshark is a critical tool in cybersecurity investigations

## Challenges Faced
- Initial difficulty understanding large volumes of packet data
- Improved analysis by applying targeted filters
- Learning to focus on relevant traffic instead of raw output

## Real-World Relevance
- Used by SOC analysts to investigate suspicious network activity
- Helps detect data leaks, malware communication, and anomalies
- Essential tool in incident response and network forensics
